02
Giải pháp nền

Speculative Decoding truyền thống

Thay vì để model lớn tự sinh từng token, dùng model nhỏ draft trước — model lớn chỉ cần verify.

Step 1
Small Draft Model
nhanh & nhỏ
draft k tokens
Step 2
Target Model
verify 1 forward pass
decision
✓ Accept
giữ lại tokens đúng
✗ Reject
fallback + regenerate

① Model nhỏ dự đoán nhanh

Draft model nhỏ hơn nhiều, sinh k token trong thời gian ngắn. Chi phí thấp.

② Model lớn verify một lần

Target model xử lý toàn bộ k token draft chỉ qua một forward pass — nhờ parallel attention.

③ Token đúng được accept

Token nào khớp với output target model sẽ được giữ lại — có thể accept nhiều token cùng lúc.

④ Token sai bị reject → fallback

Tại vị trí đầu tiên không khớp, hệ thống fallback về token của target model và tiếp tục.

⚡ Key insight: Nếu nhiều token được accept cùng lúc, tốc độ sinh có thể tăng đáng kể mà không thay đổi output chất lượng.

Cầu nối tầng thấp

D-Flash: Text Envelope Bridge

Vì không thể chia sẻ token ID hay KV cache, D-Flash truyền text thuần qua JSON envelope.

"draft_text" : "The capital of France is Paris" required
"confidence" : 0.87 optional
"max_tokens" : 8 optional
🔌 D-Flash là cầu nối text/JSON giữa hai model khác họ. Qwen chỉ gửi draft_text. Gemma E2B tự tokenize lại bằng tokenizer của chính nó rồi verify.

Đánh giá rủi ro

Rủi ro nghiên cứu

Bốn rủi ro chính cần theo dõi trong quá trình phát triển MVP.

01

Acceptance Rate Thấp

Nếu Qwen draft khác Gemma nhiều, nhiều token bị reject. Speedup không đạt.

Mitigation Đo acceptance rate theo prompt type. Điều chỉnh draft length, prompt format, normalization.
02

Overhead > Lợi Ích

Nếu draft + parse + verify tốn hơn baseline, speedup âm. Cần profiling chi tiết.

Mitigation Đo per-stage timing: draft, parse, verify, fallback. Identify bottleneck cụ thể.
03

Cross-Tokenizer Complexity

Qwen và Gemma không tương thích token ID. Text normalization có thể tạo mismatch.

Mitigation D-Flash dùng text envelope, không chia sẻ token ID. Test normalization edge cases.
04

Feature-Level Tầng Cao Phức Tạp

Hidden states của E2B và E4B chưa chắc compatible trực tiếp.

Mitigation Để high-tier làm sau MVP. Nghiên cứu adapter/EAGLE-style projection layer.
