## 6. GSM8K DFlash-R1 regression analysis
* Task80A confirms GSM8K numeric-quality pattern.
* Task80A DFlash-R1 numeric match: 24/30.
* Task80A Baseline-AR numeric match: 25/30.
* Task80A DFlash-R1 avg e2e latency: 10.49 s, and e2e tok/s: 16.08 tok/s.
* Task80A Baseline-AR avg e2e latency: 15.96 s, and e2e tok/s: 10.96 tok/s.
* Reference Task69/Task80 DFlash-R1 timing was much faster, about 2.99 s avg e2e latency and about 56.47 e2e tok/s, so the Task80A timing shift is material.
* Conclusion: Quality is stable, timing shifted materially, but DFlash-R1 remains faster than Baseline-AR in Task80A. DFlash-R1 remains a timing/runtime watch, not a confirmed regression.
